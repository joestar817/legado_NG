# 基于档案分析

专项分析默认只读取专用入口固定 `work_key`、`domain=book_scan` 的 AgentMemory，不重新读取正文，也不得跟随全局当前在读书籍切换。先查询 manifest，再按分析方向分页读取所需类型；档案证据不足时说明扫描边界并建议继续扫描，不得自动扩大正文范围。

- 雷点：遍历全部 `event`，并联读相关 `relationship`、`character`、`reader_experience`；区分客观事件与跨事件长期劝退模式。
- 感情：读取 `relationship`、`emotional_turn`、相关事件；按事件前关系、双方意图、即时心理、后续主动行为与当前变化分析。
- 主角：读取 `character`、`decision`、相关关系和事件；分析目标、主动权、人物弧、代入体验与争议选择。
- 剧情与节奏：读取 window、reader_experience、event 和 analysis；同时呈现压力累计、实际释放和期待回报。
- 结局：读取 manifest、event、character、relationship 和结局 analysis；连载中的当前冲突不能判为烂尾。

用户明确要求证据且档案缺乏证据范围时，才加载 `book_scan_facts` 并针对性补读具体章节，随后立即通过带回执的记忆写入保存。分析完成后加载 `book_scan_report`，由它一次完成所需维度的读者向判断。专项报告可以调整正文区块，但仍禁止表格、过程文本和内部 JSON，并保留下步交互。

## 雷点分析固定顺序

选择雷点分析时，必须分页读取全部 `event`，而不是只读 `critical/high` 或 Top-K；低严重度关系节点、口头原则、心理转折和缓解节点可能组合成最主要的长期劝退模式。按以下顺序聚合：

1. **可能劝退的核心原因**：3～6 条，按阅读决策影响排序；每条写“事实链 → 读者为何可能不适”，并区分普遍高压事件与偏好相关争议。
2. **已确认重大事件**：所有 `critical/high + confirmed`，按稳定 `event_key` 去重。
3. **疑似风险**：说明事件事实还缺少什么证据；不能因动机未知把已发生事实写成疑似。
4. **核心关系历史**：既往事实、作品承诺、主角何时知情及如何反应、后续实际选择、是否反复、最终后果和释放。
5. **双向关系与心路**：A 对 B、B 对 A 的意图、边界、知情、主动性、即时心理和后续关系变化。
6. **人物弧与决策争议**：成长方向、重大心理、替代方案、筹码、代价、退出权、共同受影响者是否被沟通。
7. **长期阅读体验**：反复纠缠、边界失效、伤害无相称后果、路线期待落差、持续压抑和实际缓解。
8. **已解决或反转的过程风险**：保留发生事实、持续时间和最终状态，不用结局洗掉过程。
9. **扫描边界**：真实覆盖率、未读章数和会影响结论的档案缺口。

## 其它分析方向

- **感情分析**：先给路线和核心对象，再逐条展示事件前关系、双方期待、关键变化、即时心理、后续主动行为、当前状态与边界；不把普通剧情关系写成感情线。
- **主角分析**：围绕目标、稳定特征、主动性、边界、重大选择、人物弧和代入距离；不写履历式人物介绍，不用赞美词代替判断。
- **剧情与节奏分析**：区分结构事实、角色决策逻辑、压力累计、高潮/反击、有效释放与重复模式。证据足够时允许直接判定降智、作者强行、廉价刺激或模板拼接；证据不足时先说已确认的问题，再明确缺口。
- **结局分析**：区分当前连载状态、主线已确认结局、番外/旧时间线和未收束线索；读到末尾不等于覆盖完成。

专项报告采用“主观判词 + 事实链 + 为什么烂/为什么好 + 反证或缺口”的短项结构。不要重新输出完整作品画像和人物小传，不追加宣传性总结，也不为保持平衡硬找优点。
