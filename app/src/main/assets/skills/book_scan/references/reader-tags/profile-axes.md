# 基础档案轴

基础档案轴不是作品好坏判断。感情路线和叙事中心允许多选；防御等级是唯一有序单选。字段不存在表示未知。

## 感情路线 `relationship_routes`

- `route.no_romance`｜无女主／无感情线：作品不设置核心女主，或恋爱不是持续主线。
- `route.single_partner`｜单女主／一对一：只有一名稳定核心感情对象。它只描述结构，不自动等于甜、纯爱或关系健康。
- `route.multi_partner`｜后宫／多女主：男主与多名女性建立明确感情或归宿关系；仅有多名女性 POV 或单向好感不够。

“都可以”是交互快捷项，存储为不限制，不是第四种作品路线。

## 叙事中心 `narrative_centers`

- `center.single_male`｜单男主中心：一名男性承担绝对叙事和因果中心。大量高价值配角、时代传奇或一男多女分镜都不自动算群像；《遮天》按本产品口径归入此类。
- `center.multi_male_ensemble`｜多男主真群像：多名男性主角各自拥有独立目标、主线和不可替代的因果作用，删掉任一核心线都会破坏作品主体；《紫川》按本产品口径归入此类。

“伪群像”只用于解释为什么某书仍是单男主中心，不保存为第三个值。不支持女频逆后宫档案。

## 阅读防御 `defense_level`

- `defense.god`｜神防：极端内容通常也能继续，但已确认严重风险仍完整展示。
- `defense.heavy`｜重甲：能接受较强冲突，严重关系风险仍会影响推荐。
- `defense.cloth`｜布甲：能接受轻度郁闷，明确雷点会明显影响推荐。
- `defense.low`｜低防：对压抑、关系伤害和长期吃瘪敏感。
- `defense.negative`｜负防：轻度风险也可能直接影响阅读决定。

“诡防”不是线性第六档。用户对某题材或某个标签的例外，保存为具体标签态度或自然语言规则。
