# 读者标签库 v1

本标签库把男频网文常见偏好与《清风扫书名称定义》中的有效扫雷经验整理成稳定、可扩展的 System Skill 语义。它不是首次问卷，也不是关键词黑名单。

## 规模与分层

- 3 个基础档案轴，共 10 个可选值：见[基础档案轴](profile-axes.md)。
- 5 个与基础档案轴共用稳定 ID 的中性路线、27 个通用作品标签与 11 个条件式标签：见[作品成分与可学习偏好](selectable-tags.md)。
- 11 个内置严重风险：见[内置风险](risk-tags.md)。这些风险只预警，不询问用户是否喜欢。
- 17 个深度分析信号：见[质量与长期体验信号](analysis-tags.md)。快速定位不得据此评价全书。

除 5 个复用基础档案轴 ID 的中性路线外，共有 66 个稳定作品标签；目录文件因此共有 71 条作品定义行。全量标签只供 Agent 按需读取；任何单次用户交互都不得把它们平铺成几十个选项。

## 稳定记录字段

每个条目按以下字段理解：

- `tag_id`：稳定英文 ID，写入事实或偏好后不得随显示文案改名。
- `label`：面向普通用户的中性中文名。
- `layer`：`adaptive|conditional|always_warn|analysis_only`。
- `detectability`：`high|medium|low`。
- `question_policy`：`selection_impact|never_ask`。只有前者允许进入快速定位偏好卡；缺失默认 `never_ask`。
- `definition`：什么条件下算命中。
- `exclude`：最容易发生的误判边界。
- `aliases`：只用于理解用户原话或旧社区术语，不默认显示。

`detectability` 语义：

- `high`：样本覆盖到明确设定或直接事件即可确认。
- `medium`：需要样本内重复行为或直接上下文；单一关键词不够。
- `low`：依赖中段、长期人物变化或全书结构；首尾快扫只能保持未知。

## 标签与用户态度分离

作品标签是事实，用户态度是偏好。两者不得写在同一字段里：

```json
{
  "tag_id": "setting.system_present",
  "stance": "like|neutral|avoid",
  "source_kind": "ui_choice|explicit_text",
  "source_text": "用户明确表达的原话",
  "confirmed_by_user": true
}
```

- 未保存该 `tag_id` 表示 `unknown`，不能推断为喜欢、无所谓或讨厌。
- `neutral` 是用户明确选择“无所谓”，属于已知态度，以后不重复询问。
- 跳过、关闭卡片、停止扫描、弃书、删除书籍、阅读时长和未选择均不能生成长期偏好。
- 用户只批评当前书某段写法时，默认是单书反馈；要泛化为以后同类作品的偏好，必须得到明确确认。

## 按需学习规则

快速定位先记录当前书事实，再处理偏好：

1. `always_warn`：证据为 `confirmed|ongoing` 时直接预警，不生成偏好问题。
2. `adaptive`：只表示可记录为作品事实；同时显式标记 `question_policy=selection_impact`、当前书已确认命中、用户态度仍未知、回答会实质改变推荐且证据满足检测门槛时，才有资格询问。
3. `conditional`：只有相关题材已经命中，或用户自然语言主动提到时才加载和询问。
4. `analysis_only`：只能在覆盖充分的连续扫描或深度分析中使用，不能进入快扫偏好卡。
5. 每次默认选择 3 个，只有 1～2 个时不凑数，信息确有必要时最多 5 个；跨类别去重，按“对当前推荐的决策价值”排序。
6. 用户明确反馈后只合并本次有选择的项，复用已有正文事实重算推荐，不重读正文。

负面人物判断和质量结论（例如窝囊、舔狗、降智、人设崩坏、剧情崩坏）即使能记录，也固定 `never_ask`。确定严重风险同样固定 `never_ask`；它们只进入事实、警告或深度分析，不能包装成用户爱好。

## 严重度、状态与标签不是一回事

- `god_risk|risk|frustrating` 是基础严重度，不是标签。
- “伪雷”映射为原风险仍保留且 `evidence_status=reversed`；结果未明时用 `outcome_unknown`，不能先写成已反转。
- “诡雷”只表示罕见或出乎预期，可附 `rarity=unusual`，不建立一个能掩盖具体事实的笼统标签。
- “雷／郁闷”不能代替具体 `tag_id`。

## 旧术语处理

- `推书／扫书／扫雷／求扫／神农` 是工作流用语，不入标签库。
- `仙草／粮草／干草／水草／毒草` 是作品评分词，不入标签库；“水草”的可验证含义映射 `quality.plot_padding`。
- `QC` 在原始资料中同时被用作“全初”和“全处”，禁止作为稳定 ID。
- `QCQS` 是多个条件组合的展示简称，不保存成单一事实标签。
- 带贬损性的女性称呼、成人性癖黑话只作为必要的输入别名理解，不成为默认显示名或首次交互选项。
- “脑残、开局毒点、重要女主有雷点、主角性格讨厌”等范围过宽，必须落到具体行为标签，禁止直接持久化。

## 加载路由

- 判断叙事中心、感情路线或防御字段：读取[基础档案轴](profile-axes.md)。
- 当前书命中系统、国运、直播、主角类型、后宫或特殊设定：读取[作品成分与可学习偏好](selectable-tags.md)。
- 发现背叛、第三方关系、死亡、严重虐待或强行复合：读取[内置风险](risk-tags.md)。
- 用户要求评价水文、降智、烂尾、人设崩塌等长期质量：覆盖充分后读取[质量与长期体验信号](analysis-tags.md)。
