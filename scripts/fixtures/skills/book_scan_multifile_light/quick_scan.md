# 轻量首次快速定位（实验 C：首 3 章 + 尾 3 章）

## 固定流程

1. 调用 `bookshelf_current_book_get`、`agent_memory_status_get`，再以 `scope_type=book`、当前 `work_key`、`domain=book_scan`、`memory_type=manifest` 精确查询记忆；随后调用 `bookshelf_cache_status_get`。
2. 没有 manifest 且正文可用时，完整读取物理开头 3 章：`start_chapter_index=0,chapter_count=3,char_limit=0`。
3. 完整读取物理末尾 3 章：起点 `max(0,total_count-3)`，`chapter_count=3,char_limit=0`。
4. 同时综合首尾正文，先提取人物、关系、不可逆伤害、重大关系变化、压抑和读者体验，再用一次 `agent_memory_batch_upsert` 保存 manifest、两个 window、人物、关系、阅读体验和事件记忆。
5. 记忆写入成功后直接输出完整手机报告和下一步 interaction。不要输出过程播报或内部 JSON。

每条记忆固定使用 `scope_type="book"`、`scope_key=bookshelf_current_book_get.work_key`、`domain="book_scan"`。`work_key` 必须原样传递，不能把其中的换行变成字面量 `\n`。已有记忆必须复用查询结果中的 `id`；不得维护第二套扫书档案或输出私有档案协议。

## 必须理解的数据

- 真正的主要代入角色、身份与当前处境。
- 核心关系的双方、事件前关系、感情期待和当前变化。
- 永久伤残、死亡、长期囚禁等不可逆伤害。
- 被迫婚配、背叛、下药、强迫、情感勒索等重大关系风险。
- 当事人的即时心理、选择空间和后续主动行为。
- 对读者而言是压抑、愤怒、爽点、看乐子还是证据不足。

同一事件不能脱离关系语境机械判雷；但关系前史不足也不能删除已确认的高危事实，应写“事件已确认，关系语境待补”。

### 重大风险硬约束

- 核心感情角色与主角互有情愫或已有明确感情期待，却被迫嫁给第三方：记录 `relationship_forced_marriage`，通常为 `critical`；按项目口径并列 `relationship.green_hat` 与 `relationship.sent_love_interest`。被迫、未圆房、第三方失去性能力和未来可能反转不能淡化已经发生的过程。
- 主要代入角色永久肢体伤残：记录 `character_permanent_injury`，至少为 `high`。
- AgentMemory 或报告已经包含上述事实时，禁止输出“暂未发现重大雷点”。
- 黑暗操控型主角应进入主角画像和慎入提示，不能用“这是主角特性”直接免除读者风险。
- 下药等事件如果发生在长期双向暧昧关系中，受影响者半推半就且之后主动靠近，读者反馈可能是高敏感关系推进或看乐子；仍需说明同意边界，但不能机械等同于纯伤害。
- 风险事实必须由本轮实际读取的首尾正文直接支持。禁止根据书名、常识、模型记忆或未读取章节补写番外、结局和具体事件；无法从已读正文定位的内容不得进入事件记忆。
- 死亡不能误写成永久伤残。已在主时间线彻底改写、当前没有持续伤害的前世悲剧只作为背景观察；普通黑暗背景、果断性格、正常恋爱/后宫路线和已经有效释放的旧冲突，不自动升级成重大雷点。

## 一次提交的最小记忆

至少保存这些分类型记忆：

1. 唯一 `manifest`：总章数、首尾覆盖、缺口、摘要、事件和其它记忆 id。
2. 两条 `window`：开头与结尾样本的短事实和证据范围。
3. 每名主要人物一条 `character`；每条核心关系一条 `relationship`。
4. 一条 `reader_experience`：整体风格、压抑来源、缓解和读者体验。
5. 每个已确认或疑似重大事件一条 `event`：类型、严重度、状态、人物、短事实、章节索引、关系语境和证据缺口。

结构化字段放在每条记忆的 `data` 对象中，禁止把数组或对象压成 JSON 字符串。没有风险时不创建虚假 `event`；正文明确出现高危事实时必须存在对应事件记忆。

## 手机报告

所有工具完成前，assistant 消息的 `content` 必须为空，只发起工具调用；不得输出“我先加载”“正在读取”“现在提交”等过程文本。最终只输出一次完整报告。

按以下顺序，用短段落和 Callout，任何区块都不得使用 Markdown 表格：

1. 基础信息和一句话定位。
2. 作品画像：流派、卖点、风格、世界观、结构与视角、适合/慎入读者。
3. 主要主角画像：身份、特征、代入体验和可能劝退点。
4. 核心情感关系：既有关系、当前变化和读者影响。
5. 压抑指数、压力来源和缓解边界。
6. 已发现重大雷点；`critical/high` 必须显示并写清事实与关系边界。
7. 明确仅覆盖首 3 章与尾 3 章。
8. `book_scan_next` interaction。

最终发送前只在内部做一次格式自检：删除所有过程播报；如果任意一行以 `|` 开头和结尾，必须改写为小标题或项目符号；最终回复不得出现工具名、记忆字段名、JSON schema 或 Provider 协议标记。自检过程本身不要输出。

没有重大风险时只给中性结论，不列举本轮未命中的绿帽、强制婚配、下药等具体术语清单。
